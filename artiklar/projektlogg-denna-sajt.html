Projektlogg: så byggde jag den här sajten säkert

Den här sajten är medvetet enkel — och det är ett säkerhetsval i sig. Här går jag igenom hur den är byggd och varför, för dig som är nyfiken på hur säkerhetstänk kan se ut i praktiken redan från grunden.

Statisk sajt = mindre angreppsyta

Sajten är byggd med ren HTML, CSS och lite JavaScript och ligger på GitHub Pages. Det finns ingen databas, inget inloggningssystem och ingen serverkod som kan utnyttjas. Färre rörliga delar betyder helt enkelt färre saker som kan gå sönder eller missbrukas.

HTTPS från start

All trafik till och från besökare är krypterad via HTTPS. Det är i dag en självklarhet snarare än ett extra steg — men värt att alltid kontrollera att det faktiskt är påslaget.

Inga onödiga beroenden

Stora ramverk och tredjepartsskript är medvetet bortvalda. Varje extern kodbit man laddar in är något man måste lita på. Här laddas bara typsnitt från en känd källa — inget mer. Färre beroenden, mindre risk.

Principen: Säkerhet handlar ofta om att ta bort saker, inte lägga till. Det enklaste systemet som löser uppgiften är oftast också det säkraste.

Samma råd jag ger till kunder

Att hålla saker enkla, förstå vad varje del gör och välja beprövade lösningar framför komplicerade genvägar — det är samma grundregler som gäller för en webbplats som för en hel verksamhet. Säkerhet behöver inte vara avancerat för att vara effektivt.

Hör av dig så pratar vi om din situation – helt förutsättningslöst.
